What is a null-terminated String?</h2></div> <div><h3>Answer:</h3> <div class="answer"><div><div><div class="AnswerBody"><p>A "string" is really just an array of <code>char</code>s; a null-terminated string is one where a null character <code>'\0'</code> marks the end of the string (not necessarily the end of the array). All strings in code (delimited by double quotes <code>""</code>) are automatically null-terminated by the compiler.</p><p>So for example, <code>"hi"</code> is the same as <code>{'h', 'i', '\0'}</code>.</p><p>Null-terminated strings are often a <em>drain on performance</em>, for the obvious reason that the time taken to discover the length depends on the length. The usual solution is to do both - keep the length and maintain the null terminator. It's not much extra work and means that you are always ready to pass the string to any function.</p></div></div><div class="row my-2"><div><span><i>Source:</i>&nbsp;<span><a href="https://stackoverflow.com/questions/2037209/what-is-a-null-terminated-string" rel="noreferrer" target="_blank" title="What is a null-terminated String?
